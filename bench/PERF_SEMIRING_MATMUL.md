## Interpretation

- The reported wall-time is the *median* of timed runs after
  ``warmup`` un-timed calls; first-call compile cost is in the
  separate compile-time table.  Compile times above ~5 s on
  cold cache should be reproducible across re-runs in the same
  process (JAX caches the lowered HLO; Pallas caches the
  Triton lowering).
- Speed-up < 1× on a given algebra is a signal the Pallas
  kernel needs tile / unroll re-tuning -- file as an issue
  against the kernel, not as a missing feature.
- For ``REAL``, the more relevant baseline is ``jnp.matmul``
  with default precision (tensor cores).  This bench compares
  against the same-algebra reference (CUDA cores) so the
  numbers are apples-to-apples with the other algebras.
